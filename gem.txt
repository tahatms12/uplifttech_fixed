1. Eliminate 404s. Either publish the missing routes (/about, /careers, /case‑studies, /contact, /privacy‑policy, /services, /llms.txt) or strip them from sitemap.xml. A sitemap must never reference a non‑200 URL.

2. Fix sitemap.xml. List only canonical HTTPS URLs. Regenerate after every deployment. Add to robots.txt:

```
Sitemap: https://uplift‑technologies.com/sitemap.xml
```

3. Consolidate hostnames. Force one scheme and one subdomain. Example Netlify \_redirects rule:

```
http://*  https://uplift‑technologies.com/:splat  301!
https://www.uplift‑technologies.com/*  https://uplift‑technologies.com/:splat  301!
```

4. Robots.txt. Minimal safe version:

```
User-agent: *
Allow: /
Sitemap: https://uplift‑technologies.com/sitemap.xml
```

5. Head section template. Place in every HTML file or React layout:

```html
<title>%%PAGE_TITLE%% | Uplift Technologies</title>
<link rel="canonical" href="https://uplift‑technologies.com%%PATHNAME%%">
<meta name="description" content="%%120‑160 CHAR SUMMARY%%">

<!-- Open Graph -->
<meta property="og:type" content="website">
<meta property="og:url" content="https://uplift‑technologies.com%%PATHNAME%%">
<meta property="og:title" content="%%PAGE_TITLE%%">
<meta property="og:description" content="%%120‑160 CHAR SUMMARY%%">
<meta property="og:image" content="https://uplift‑technologies.com/assets/og/%%SLUG%%.jpg">

<!-- Twitter Cards -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="%%PAGE_TITLE%%">
<meta name="twitter:description" content="%%120‑160 CHAR SUMMARY%%">
<meta name="twitter:image" content="https://uplift‑technologies.com/assets/og/%%SLUG%%.jpg">

<script type="application/ld+json">
{
  "@context":"https://schema.org",
  "@type":"WebPage",
  "url":"https://uplift‑technologies.com%%PATHNAME%%",
  "name":"%%PAGE_TITLE%%",
  "description":"%%120‑160 CHAR SUMMARY%%",
  "inLanguage":"en"
}
</script>
```

6. Page titles. One unique, keyword‑front‑loaded `<title>` ≤60 characters per page. Example: “AI‑Driven Freight Dispatch | Uplift Technologies”.

7. One `<h1>` per page mirroring the title. Subsequent headings in hierarchical order.

8. Asset weight. Inline critical CSS (<15 KB), lazy‑load images, serve AVIF/WebP, add `fetchpriority="high"` to the hero image. At build time run:

```
bun run vite build --minify --sourcemap=false
```

Then gzip or br brotli compress assets and set `Cache-Control: public,max-age=31536000,immutable`.

9. Core Web Vitals. Target:
   LCP <2.5 s. INP <200 ms. CLS <0.1. Remove render‑blocking JS. Use `load="async"` for analytics.

10. HTML validity. Pass every page through [https://validator.w3.org/nu](https://validator.w3.org/nu).

11. Internal links. Every page must be reachable in ≤3 clicks. Descriptive anchor text, no “click here”.

12. Image alt text. Filename and alt both include primary keyword.

13. RSS feed. Static `/feed.xml` announcing new content with `<lastBuildDate>` updated on deploy for faster discovery.

14. HTTP status hygiene. 200 for live. 301 for moves. 410 for permanently removed. Never leave a 404 in production.

15. Deploy, crawl with Screaming Frog or `wget --spider`, verify zero 404, one 301 chain max, metadata coverage 100 percent. Terminate.
